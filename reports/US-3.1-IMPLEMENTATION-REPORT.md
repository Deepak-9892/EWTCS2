## Overview

Implemented server-side capture of patient admission time when a bed transitions from a non-patient stage (Empty/Cleaning) to an occupied stage. The timestamp is generated by the database server, stored as a timezone-aware value, and remains immutable until the bed is cleared back to a non-patient stage.

## Implementation Details

### 1. Server-Side Mutation Logic

**File**: `src/features/bed-dashboard/lib/bed-mutations.ts`

- Uses `NOW()` in SQL to set admission time when a bed becomes occupied.
- Keeps `patient_start_time` unchanged if already set.
- Clears `patient_start_time` only when bed returns to non-patient stages (Empty/Cleaning).
- Returns the stored timestamp from the database after update.

**Core SQL logic:**
```
patient_start_time = CASE
  WHEN $2 THEN NULL
  WHEN patient_start_time IS NULL THEN NOW()
  ELSE patient_start_time
END
```
